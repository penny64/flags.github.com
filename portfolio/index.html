First...

I've been developing games in some capacity for about ten years, getting my start as early as elementary school. Only recently have I decided to take it on as a career, and this summer I will be working on publishing my first major game - "Torch!"

Roguelikes are a genre of games that focus on a core set of features. I find these parameters both fun and challenging to work in, and have adopted them into most of my projects.

  • Randomly generated levels. Each time you play the game you travel through a different world, with different items, randomized enemies, and traps. No experience is the same as the last. This creates an incentive to play the game many times over.
  • Like Dungeons and Dragons, the player is able to build a unique character with its own strengths and weaknesses. The catch: once their character dies, they are gone forever. Hours of progress can be lost with a single mistake - however, many people prefer this brutal spike in difficulty.

With these things in mind, I've set out to create games that are both extremely challenging and rewarding. And despite not having much in the way of art, I work in a very minimal (but visually pleasing) style that uses only letters and symbols to build lush, rich worlds.

Torch!

A coffee-break roguelike/RPG designed to place emphasis on player movement and positioning. Features a unique combat system and randomly generated levels, puzzles, and items for near-infinite replayability. Written in C using the libTCOD graphics library.

  • Unique weapons: Daggers, spears, and swords each require the player to approach combat in a different way.
  • Not your average roguelike. Torch focuses on building brightly-colored levels that are as entertaining to traverse as they are to look at.

GOAPy

GitHub - Example

GOAPy is a pure-Python implementation of Goal-Oriented Action Planning, a generic architecture for autonomous AI.

By taking the current state of the AI (e.g.: Hungry? Tired?) and their desired state (e.g.: Not tired) a "plan" is generated to take the AI from their current state to their goal state by considering a list of actions. In this case, the AI will discover that in order to sleep, they must first find and consume food, potentially leading to a complex course of action that involves going shopping, cooking, ordering pizza, taking a shower, etc.


Insurgence: Shadow Operation

GitHub

Prototype developed to experiment with a mix of turn-based strategy, squad-based combat AI, and procedural building generation. Developed using the second iteration of a custom Entity Component System.

  • Modular building generation: Given a list of rooms and their attributes, an entire floor-plan is generated and constructed, then rendered in-game.
  • Adaptive Enemy AI: Using GOAPy, AI squad commanders can position their units in strategically-critical positions, evade attacks, or ambush the player.
  • Asymmetric multiprocessing: CPU-heavy operations can be offloaded to a secondary process, allowing graphics rendering to continue while numbers are being crunched.

Reactor 3 - The Dead World

GitHub

Open-world survival simulator set in a post-apocalyptic Europe. Features complex NPC behavior, detailed damage model, and procedurally generated assets. Developed for two years using Python in addition to Cython-compiled modules.

  • Living, breathing world: NPCs loot, manage items, and participate in a large-scale battle between factions.
  • Day/night cycle with weather effects.
  • Particle system capable of dynamic lighting.
  • Huge open-world filled with towns, military outposts, and other landmarks.
  • Basic mod support - create items and lifeforms to inhabit the world and script their behaviors.
